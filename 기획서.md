## 3. 데이터 연동 설계 (기능 단위)

### F1. 구글 시트 실시간 로드
- **라이브러리:** PapaParse (CDN) — 값에 쉼표·따옴표가 있어도 안전하게 파싱.
- **흐름:**
  1. 페이지 로드 → `Papa.parse(SHEET_CSV_URL, { download: true, header: true, skipEmptyLines: true })`
  2. `complete` 콜백에서 `results.data`(객체 배열, 키=컬럼명)를 전역 상태에 저장.
  3. 각 행을 정규화(`trim`, 빈 값 처리) 후 렌더.
- **URL 상수 (app.js):**
  ```js
  const SHEET_CSV_URL =
    "https://docs.google.com/spreadsheets/d/1qSz3fbUnOspUyiB5wYvpJt6vtxTlRt1yAQ_VXjewLfo/gviz/tq?tqx=out:csv&sheet=data&headers=1";
  ```
- **상태 처리:** 로딩 스켈레톤 → 성공 시 목록, 실패 시 안내 메시지("연수 정보를 불러오지 못했습니다. 잠시 후 새로고침해 주세요.").
- **실시간 반영 검증:** 시트 한 칸 수정 → 사이트 새로고침 시 반영 확인(Step 5).

### F2. 검색 (메인)
- 상단 검색창 입력값으로 **과정명 · 한줄소개 · 주요내용 · 담당부서**를 대소문자·공백 무시 부분일치 필터.
- 입력 즉시(debounce ~150ms) 목록 갱신, 결과 수 표시("N개 과정").

### F3. 필터 (메인)
- **분야**(단일/다중 택1 토글 칩), **이수구분**, **교육방식**, **대상**을 조합(AND) 적용.
- 검색어 + 필터는 동시에 AND로 결합.
- "전체" 초기화 버튼. 결과 0건 시 빈 상태 메시지.
- (선택) 현재 필터를 URL 쿼리에 반영해 공유·뒤로가기 지원.

### F4. 메인→상세 연결
- 카드 클릭 → `detail.html?code=<과정코드>`로 이동.
- 상세 페이지는 같은 시트를 로드한 뒤 `code`와 일치하는 과정 1건을 찾아 렌더. (없으면 "과정을 찾을 수 없습니다" + 메인 복귀 링크)

## 6. 디자인 방향 (frontend-design 기반 — 초안, Step 3에서 확정)

> 브리프: 부산대 교직원이 보는 **연수 편람**. AI 기본값(크림+세리프+테라코타 / 블랙+형광 / 신문 조판)은 피하고, "학사 편람의 질서 + 코드·라벨 시스템"이라는 방향으로 설계한다.

- **팔레트(4–6):**
  - `--ink` `#12263A` (부산대 계열 딥 네이비 — 제목·강조)
  - `--paper` `#F7F5F0` (차분한 웜 화이트 배경)
  - `--text` `#1C1E22` (본문 먹색)
  - `--line` `#DAD5C9` (헤어라인 구분선)
  - `--accent` `#0B6E6E` (딥 틸 — 링크·포커스·강조 포인트)
  - 이수구분 신호색: 필수=진한 네이비 솔리드 · 권장=틸 아웃라인 · 선택=중립 그레이
- **타이포:** 코드/데이터는 모노가 실제로 "코드"라는 점을 살린다.
  - 디스플레이: `Noto Serif KR` (제목에 학술적 무게)
  - 본문: `IBM Plex Sans KR` (행정 문서의 명료함)
  - 데이터/과정코드: `IBM Plex Mono` (PNU-2026-XXX)
  - ※ 모두 Google Fonts CDN → GitHub Pages에서 로드 가능
- **레이아웃:** 좌측에 **분야별 색 스파인(spine)**을 둔 카드 그리드. 카드 상단에 모노 과정코드 + 이수구분 배지.
- **시그니처:** 10개 **분야를 색으로 코드화**하여 메인 카드 → 상세 배지까지 일관되게 흐르게 하는 "색인(index) 시스템". 히어로는 실시간 집계 지표 + 즉시 검색.
- **품질 기준:** 모바일 반응형, 키보드 포커스 가시화, `prefers-reduced-motion` 존중.
